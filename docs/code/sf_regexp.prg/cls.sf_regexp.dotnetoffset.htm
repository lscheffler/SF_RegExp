DotNetOffset Property
Class: SF_RegExp
This controls if some data start with 0 or 1.
Object.DotNetOffset
=
lvVal
Value
lvVal
Type Boolean
Direction In- and Output
Remarks

Some of the DotNet operations start with index or position 0. To make the use normal for VFP they default to start with 1. If 0 should be used, it is possible to set this here.

  • IsMatch (2nd parameter in the StartAt use)
  • Match (2nd parameter in the StartAt use)
  • Matches (2nd parameter in the StartAt use)
  • Replace (4th parameter in the StartAt use)
  • Split (2nd parameter in the StartAt use)
  • Index Property of Match, Group and Captures objecs via Get_Index

There is a Assign controling the use.
