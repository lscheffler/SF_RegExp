Class: SF_RegExp

A class as interface to DotNet RegExp.
Remarks

This class is set up to create an interface to the DotNet RegExp Namespace. It uses SF_RegExp.dll and wwDotNetBridge to acces the RegExp.

The document does not contain help how to use the DotNet RegExp, see there. As far as possible the methods and properites of RegExp, Matches, Match, Groups, Group, Captures and Capture are reflected in the class and the return of methods. The Options property of the RegExp is split into several single booleans for better use with VFP. See

  • Compiled
  • CultureInvariant
  • ECMAScript
  • ExplicitCapture
  • IgnoreCase
  • IgnorePatternWhitespace
  • MultiLine
  • RightToLeft
  • SingleLine

The return of the Matches and Match method may be a DotNet object or a VFP object, see ReturnFoxObjects. The depth of the resolution is controled by the AutoExpandGroups and AutoExpandCaptures properties, this allowing to only transform data as needed.
Note, this changes the way collections (Matches, Groups, Captures) are enumerated. DotNet (and VBasic) start with 0, VFP with 1.

Some example to use the class are available in the example program.

For the leisure of VFP programmers, strings start with 1 for this class in all methods and properties dealing with positions in strings. If you like it with 0, set DotNetOffset.

For easy reuse of VBasic Regexp, a Execute method is provided. The Match objects own a FirstIndex property that works like VBasic. Note that the Matches collection returned now starts with 1.
