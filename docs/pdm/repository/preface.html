Purpose

The goal of this tool is to access the DotNet RegExp Namespace with VFP.

The usual RegExp in VFP is that of VBasic. It's one of the most limited ways to use RegExp. For GoFish there was a need to do a more compley match then VBasic allowed, so here is the result.

The access runs through wwDotNetBridge. The class is set up to hide to access.

Since there was a problem to access the overloaded DotNet methods, an additional C# wrapper is used for access.

The return of the wwDotNetBridge calls are DotNet objects. This class allows to simply proceed with this objects and offers access methods to the serveral properties. Or the return is a structure made out of VFP objects. It's basically a matter of speed what to use, the full VFP version will expose mostly all properties what needs time to load, the other needs more code to get the property values in the code running, but could be limited to minimum. See ReturnFoxObjects. See there for the options to get finer control over the return.

Usage

The document does not contain help how to use the DotNet RegExp Namespace, see there. As far as possible the methods and properites of RegExp, Matches Match Groups, Group, Captures and Capture are reflected in the class and the of methods. The Options property of the RegExp is split into several single booleans for better use with VFP.

The return value of the program is a refernce to the class instantiated.

To help to replace old VBasic based code, a method Execute is added. This returns a VFP collection like VBasic, with the exception that the collection starts with 1 not 0. For most easy use Match objects returned by this method own a FirstIndex property wotking with string start 0 as the VBasic objects. Note that all other methods do not own this property. It is recommended to rework to the native calls.
